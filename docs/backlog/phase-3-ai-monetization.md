## 3.1 AI-матчинг через OpenAI Embeddings

**Цель:** заменить взвешенный алгоритм на семантический поиск по профилям.

- [ ] Добавить поле `embedding vector(1536)` в модель `User`
- [ ] `EmbeddingService` (`app/services/embedding_service.rb`):
  - Составляет текстовое описание профиля из всех полей квиза
  - Вызывает OpenAI embeddings API (`text-embedding-3-small`)
  - Сохраняет вектор в `users.embedding`
- [ ] Обновление embedding при изменении профиля (after_save callback → job)
- [ ] Обновить `PodMatcher`:
  - cosine similarity через `neighbor` gem (pgvector)
  - Гео-фильтр остаётся (30 мин езды)
  - Similarity score заменяет ручные веса
- [ ] A/B тест: сравнить качество матчинга до/после embeddings
- [ ] Rake task для генерации embeddings для всех существующих пользователей

## 3.2 AI-предложения активностей

**Цель:** GPT-4o-mini предлагает активности под конкретную группу.

- [ ] `ActivitySuggester` сервис (`app/services/activity_suggester.rb`):
  - Собирает контекст: город, район, интересы группы, прошлые активности, текущая фаза
  - Запрос к GPT-4o-mini: 3 идеи активностей с названием, местом, описанием
  - Кэширование ответов (Rails.cache, 24 часа)
- [ ] В Pod Dashboard: секция "Идеи для вас" — 3 карточки активностей от AI
  - Кнопка "Предложить группе" — создаёт Activity из карточки
  - Кнопка "Обновить идеи" (rate-limited: раз в 24 часа)
- [ ] Персонализированные карточки вопросов:
  - GPT генерирует вопросы на основе профилей участников Pod
  - Отображаются в чате в фазе 3–4
