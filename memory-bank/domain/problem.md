# Project Problem Statement

Этот документ фиксирует общий продуктовый контекст Orbit. Feature-документы должны ссылаться на него, а не переписывать один и тот же background каждый раз.

## Product Context

Orbit помогает человеку найти "своих людей рядом" и попасть не в абстрактную большую social graph, а в маленькую локальную группу, с которой реально можно начать общаться и встречаться. Продукт строится вокруг идеи, что дружеские связи проще запускать в совместимой группе из нескольких человек, чем один на один или в шумном бесконечном фиде.

Основной пользователь Orbit регистрируется, проходит onboarding, указывает социальные предпочтения, ритм общения, интересы и географическую привязку, после чего попадает в matching pool. Алгоритм подбирает совместимую группу из пяти человек с учетом расстояния и совместимости профилей, а затем создает `Pod`, который становится основной единицей дальнейшего опыта.

После формирования `Pod` продукт смещается из стадии acquisition в стадию удержания и социального взаимодействия: участники видят состав группы, предлагают активности и общаются в pod chat. Orbit не пытается быть универсальной социальной сетью; его продуктовая граница уже и конкретнее: довести пользователя от намерения познакомиться до устойчивой небольшой группы поблизости.

## Core Workflows

- `WF-01` Регистрация -> onboarding -> заполнение профиля и геоданных -> попадание в matching pool.
- `WF-02` Batch matching -> формирование `Pod` -> нотификация пользователю -> первый вход в pod experience.
- `WF-03` Жизнь внутри `Pod` -> просмотр участников -> предложение активностей -> общение в pod chat.

## Constraints

- `PCON-01` В matching допускаются только пользователи с завершенным onboarding и заполненными координатами.
- `PCON-02` Базовая единица группировки - `Pod` из пяти человек; если подходящей пятерки нет, система может создать неполный или inactive pod, но не должна нарушать правила membership.
- `PCON-03` Пользователь не должен одновременно попадать в несколько `Pod`; downstream-фичи обязаны уважать уникальность активного membership.
- `PCON-04` Matching зависит одновременно от географической близости и совместимости профиля; feature-изменения не должны оптимизировать только один сигнал, ломая другой.
