# Brief Review

Проверь `feature.md` на готовность перейти из стадии **Draft** к работе над Design Ready (Spec).

Прочитай:
- `memory-bank/features/FT-XXX/feature.md` — canonical feature-документ
- `memory-bank/flows/feature-flow.md` — gate «Draft → Design Ready»

## Критерии проверки

Для каждого критерия дай оценку `pass` или `fail`:

1. **Проблема понятна** — секция `Problem` описывает конкретную боль или opportunity, без архитектурных решений.
2. **Scope задан** — есть хотя бы один `REQ-*`, который можно проверить.
3. **Non-scope задан** — есть хотя бы один `NS-*`, явно ограничивающий scope.
4. **Нет двусмысленности** — нет слов типа «быстро», «удобно», «при необходимости», «и т.д.».
5. **Без implementation details** — секция `Problem` не уходит в how, только в what и why.

## Что нужно сделать

1. Проведи ревью по критериям выше.
2. Для каждого `fail`:
   - приведи проблемный фрагмент из `feature.md`;
   - объясни, почему это мешает следующему этапу;
   - предложи конкретное исправление.
3. Внеси правки напрямую в `feature.md`.
4. Задай список уточняющих вопросов о том, что ещё стоит уточнить, прежде чем переходить к Spec. Для каждого вопроса сразу предложи вариант по умолчанию.

## Формат ответа

- Список замечаний (или `0 замечаний, brief готов к Spec`).
- Список уточняющих вопросов с рекомендуемым вариантом.
