# Brief Review

В концепции реализации задач через процесс `Brief -> Spec -> Implementation Plan` сделай ревью brief так, чтобы он соответствовал главным критериям:

- однозначность;
- полнота;
- непротиворечивость;
- верифицируемость.

Вот brief:

```md
[вставь сюда текст brief]
```

## Что нужно сделать

1. Проведи ревью brief по критериям выше.
2. Для каждого замечания:
   - приведи проблемный фрагмент;
   - объясни, почему это мешает дальнейшей работе;
   - предложи конкретное исправление.
3. Затем внеси изменения в brief и покажи улучшенную версию целиком.
4. После этого задай мне списком вопросы о том, что еще стоит обновить в brief с учетом замечаний.
5. В каждом вопросе сразу предлагай самое подходящее решение по умолчанию.

## Формат ответа

- Сначала список замечаний.
- Затем исправленная версия brief.
- Затем список уточняющих вопросов с рекомендуемым вариантом.

Если замечаний нет, так и напиши: `0 замечаний, brief готов к следующему этапу`.
