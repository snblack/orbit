# Implementation Plan Realization

Реализуй следующий `Implementation Plan`.

Вот план:

```md
[вставь сюда текст implementation plan]
```

## Правила выполнения

- Следуй шагам плана по порядку, если из самого плана явно не следует безопасная параллелизация.
- Не меняй scope задачи без явного основания в brief или spec.
- Если в ходе реализации обнаружится противоречие, пропущенный шаг или неверный grounding, остановись и сначала опиши проблему.
- После реализации перечисли:
  - какие шаги выполнены;
  - какие файлы изменены;
  - какие проверки или тесты запущены;
  - какие ограничения, риски или follow-up остались.
