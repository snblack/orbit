# Implementation Plan Do

Сделай, пожалуйста, `Implementation Plan` в концепции выполнения задачи, когда мы создаем 3 документа для реализации задачи: `Brief -> Spec -> Implementation Plan`.


Вот spec:

```md
[вставь сюда spec]
```

## Основные критерии плана

1. Каждый шаг конкретен — понятно, какой файл затрагивается и что именно меняется.
2. Порядок шагов корректен — зависимости не нарушены, нет циклов.
3. Каждый шаг атомарен — можно выполнить и проверить независимо.
4. Нет пропущенных шагов — например, миграции, обновление тестов, обновление документации.
5. Есть grounding — все упомянутые файлы и модули существуют в проекте.

## Что нужно сделать

- Построй план только на основе brief, spec и реального состояния репозитория.
- Перед составлением плана проверь, какие файлы, модули и паттерны уже существуют.
- Для каждого шага укажи:
  - цель шага;
  - конкретные файлы или модули;
  - что именно меняется;
  - как проверить завершение шага.
- Отдельно укажи шаги по тестам, миграциям, конфигурации и документации, если они нужны.
- Если для корректного плана не хватает данных, сначала перечисли короткие уточняющие вопросы.

## Ожидаемый результат

Верни готовый `Implementation Plan` в Markdown, пригодный для сохранения в репозитории.
